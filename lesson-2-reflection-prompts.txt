What happens when you initialize a repository? Why do you need to do it?

Git create the subdirectory .git to save all metadata of you project files. We
need it to start to use Git

How is the staging area different from the working directory and the repository?
What value do you think it offers?

The staging area is between working directory and the repository and it allows you
to add on it only the working directory files you want to make a commit in the
repository

How can you use the staging area to make sure you have one commit per logical
change?

Adding the files of the logical change to staging area and then commit

What is the result of merging two branches together? Why do we represent it in
the diagram the way we do?

The result is a new branch with all the commits and a new version of the files
with the content of the two branches. Because the new branch have all the
previous commits
